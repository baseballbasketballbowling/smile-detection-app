# 笑顔検出Webアプリ (Smile Detection Web App)

## プロジェクト概要

Claude Haiku Vision APIを使い、Webカメラ映像からリアルタイムで笑顔を検出し、最高の笑顔を自動シャッターで撮影するWebアプリケーション。旅館・婚礼会場・居酒屋での商用展開を目指す。

## 技術スタック

- **フロントエンド**: HTML / CSS / JavaScript (Vanilla)
- **笑顔検出**: Claude Haiku Vision API (claude-haiku-4-5-20251001)
- **カメラ入力**: MediaDevices API (`getUserMedia`)
- **描画**: Canvas API
- **動作確認環境**: CodePen

## 実装済みの仕様

### 5フレームバッファ方式
- カメラ映像を一定間隔でキャプチャし、直近5フレームをバッファに保持
- バッファ内のフレームをHaiku Vision APIに送信して笑顔スコアを取得
- スコアが閾値を超えたフレームを「ベストショット」として保存

### 笑顔検出フロー
```
カメラ映像 → フレームキャプチャ → base64エンコード
→ Haiku Vision API (笑顔スコア判定) → スコア閾値チェック
→ 自動シャッター（画像保存）
```

## 既知の問題

### シャッタータイムラグ（最優先課題）
- **現象**: 笑顔を検出してからシャッターが切れるまでにラグがある
- **原因候補**:
  1. Haiku Vision APIのレスポンス遅延
  2. base64エンコード・デコードの処理時間
  3. フレームバッファの評価タイミング
- **改善方向**:
  - APIリクエストの並列化
  - バッファ評価ロジックの最適化（スコアが下がり始めたら直前フレームをシャッター）
  - フレームキャプチャ間隔の調整

## API設定

- **使用モデル**: `claude-haiku-4-5-20251001`（最新Haikuを常に使用）
- **APIキー**: 利用者が別途保有（アプリ内で入力 or 環境変数で設定）
- **エンドポイント**: `https://api.anthropic.com/v1/messages`
